    **Create `apps/api/src/utils/deidentify.ts`** — Pure TypeScript, no external dependencies.

    This utility implements HIPAA Safe Harbor de-identification per 45 CFR 164.514(b)(2)(i). It must include:

    1. **PHI_PATTERNS constant** — Regex patterns covering the 18 Safe Harbor identifier categories:
       - A. Names (not feasible via regex alone — handled by field stripping, not text scrubbing)
       - B. Geographic: ZIP codes `/\b\d{5}(?:-\d{4})?\b/g`
       - C. Dates (except year): `/\b\d{1,2}[\/\-]\d{1,2}[\/\-]\d{2,4}\b/g` and month-name patterns
       - D. Phone: `/\b(\+?1[-.\s]?)?\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}\b/g`
       - E. Fax: same pattern as phone (contextual, caught by phone pattern)
       - F. Email: `/\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b/g`
       - G. SSN: `/\b\d{3}[-\s]?\d{2}[-\s]?\d{4}\b/g`
       - H. Medical record numbers: `/\b[A-Z]{2,3}-?\d{4,10}\b/g`
       - I-K. Beneficiary/account/license numbers: covered by generic alphanumeric patterns
       - L-M. Vehicle/device identifiers: context-specific, documented as field-level stripping
       - N. URLs: `/https?:\/\/[^\s]+/g`
       - O. IP addresses: `/\b(?:\d{1,3}\.){3}\d{1,3}\b/g`
       - P-Q. Biometric/photos: handled by not sending these fields, not by regex
       - R. Other unique identifiers: userId, database IDs — handled by field-level stripping

    2. **`scrubPhiFromText(text: string): string`** — Replaces PHI patterns with `[REDACTED]`. For scrubbing free-text strings before sending to AI.

    3. **`deidentifyUserForAI(user: { dateOfBirth?: Date | null; biologicalSex?: string | null }): SafeUserContext`** — Strips direct identifiers from a user object. Returns only Safe Harbor-safe fields: birthYear (year only), ageRange (decade bucket, "90+" for ages > 89), biologicalSex. Explicitly does NOT include userId, email, name, zip, phone, dob (exact), or any other identifier.

    4. **`assertNoPhi(text: string, context: string): void`** — Throws an Error if any PHI pattern is detected in the text. Use as a runtime guard before AI calls.

    5. **Export types:** `SafeUserContext`, `DeidentifiedContext` interfaces.

    Include JSDoc comments referencing 45 CFR 164.514(b)(2)(i) for each function. Use `.js` extension in imports (ESM compatibility per codebase convention).
