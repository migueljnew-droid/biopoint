    The user must complete these administrative steps in vendor dashboards. Claude cannot perform these actions (they require authenticated vendor portal access and legal agreement signing).

    **Step 1 — Neon BAA (COMP-01):**
    1. Log in to Neon Console (console.neon.tech)
    2. If not on Scale plan: Billing -> Change Plan -> Scale
    3. Organization Settings -> HIPAA support -> Enable -> Accept BAA terms
    4. Project Settings (production project) -> HIPAA support -> Enable
       - WARNING: This restarts all computes. Do during low-traffic window.
       - WARNING: This is irreversible per project.
    5. Save/screenshot the BAA confirmation page

    **Step 2 — Cloudflare R2 Decision (COMP-02):**
    1. Email enterprise@cloudflare.com:
       Subject: "Does Cloudflare Enterprise BAA explicitly cover R2 object storage for HIPAA PHI?"
       Body: "We are a healthcare app storing medical images in R2. We need written confirmation that R2 object storage is within scope of the Cloudflare Enterprise BAA before we can proceed."
    2. Set a 5-business-day deadline for response
    3. **If Cloudflare confirms R2 BAA coverage:** Sign Enterprise contract + BAA
    4. **If Cloudflare cannot confirm (or no response in 5 days):** Migrate to AWS S3:
       a. Create S3 bucket "biopoint-uploads" in us-east-1 (AWS Console -> S3 -> Create Bucket)
       b. Enable default encryption (SSE-S3 or SSE-KMS)
       c. Block all public access
       d. Create IAM user with S3-only policy for the bucket
       e. Accept AWS BAA: AWS Console -> AWS Artifact -> Agreements -> Accept
       f. Update env vars: remove S3_ENDPOINT, set AWS_REGION=us-east-1, update AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY with new IAM credentials

    **Step 3 — Render BAA (COMP-03):**
    1. Log in to Render Dashboard
    2. Ensure ALL services in workspace are on paid plans (free instances will be suspended)
    3. Workspace Settings -> Compliance -> Get Started
    4. Complete the BAA signing flow (electronic signature)
    5. Click "Enable HIPAA" (or wait 72 hours for auto-enable)
       - WARNING: Irreversible. $250/month minimum + 20% usage surcharge.
       - WARNING: All services will be redeployed to HIPAA-compliant infrastructure.
    6. Save/screenshot the HIPAA workspace confirmation

    **Step 4 — OpenAI BAA (recommended, not blocking):**
    1. Email baa@openai.com:
       Subject: "BAA Request - BioPoint Health API (healthcare app)"

    2. After approval (1-2 business days): Organization -> Data Controls -> Enable Zero Data Retention

    - Change the S3Client constructor to conditionally use endpoint only if S3_ENDPOINT env var exists:
      ```typescript
      const s3Client = new S3Client({
          region: process.env.AWS_REGION || 'us-east-1',
          ...(process.env.S3_ENDPOINT ? { endpoint: process.env.S3_ENDPOINT } : {}),
          credentials: {
              accessKeyId: process.env.AWS_ACCESS_KEY_ID || '',
              secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY || '',
          },
      });
      ```
