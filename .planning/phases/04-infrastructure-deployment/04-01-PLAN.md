**Step 1: Add `directUrl` to schema.prisma datasource block**

In `/Users/GRAMMY/biopoint/db/prisma/schema.prisma`, update the datasource block from:
```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
  // Connection pool configuration comments...
}
```
To:
```prisma
datasource db {
  provider   = "postgresql"
  url        = env("DATABASE_URL")
  directUrl  = env("DIRECT_URL")
  // Connection pool configuration: DATABASE_URL = pooled (PgBouncer), DIRECT_URL = direct (no pooler)
  // Production: 20 connections (max_pool_size=20)
  // Staging: 10 connections (max_pool_size=10)
  // Development: 5 connections (max_pool_size=5)
}
```

**Step 2: Add composite index to LabMarker model**

In the `LabMarker` model (around line 195), add two new indexes at the bottom of the model alongside the existing `@@index` entries:

```prisma
model LabMarker {
  // ... all existing fields unchanged ...

  // Encryption metadata
  encryption_version   Int?      @default(1)
  encryption_metadata  Json?     @default("{}")

  @@index([value_encrypted])
  @@index([encryption_version])
  @@index([userId])                   // range scan by user (eliminates full table scan)
  @@index([userId, recordedAt])       // compound for sorted trends query: WHERE userId ORDER BY recordedAt
}
```

**Step 3: Generate Prisma migration**

Since `DIRECT_URL` may not be set in local dev environment (it's a new env var), set a placeholder for migration generation only, or use the existing `DATABASE_URL` as the direct URL temporarily to generate the migration file:

```bash
cd /Users/GRAMMY/biopoint/db

# If DATABASE_URL is set to a Neon direct URL (no pooler suffix), just generate:
DIRECT_URL="${DATABASE_URL}" npx prisma migrate dev --name add_lab_marker_trends_index --create-only
```

The `--create-only` flag generates the migration SQL without running it (safe for local generation when DIRECT_URL not configured). The migration will be applied in production via `prisma migrate deploy`.

If the local Prisma DB is not a Neon URL, also acceptable to use:
```bash
DIRECT_URL="${DATABASE_URL:-postgresql://localhost:5432/biopoint_dev}" npx prisma migrate dev --name add_lab_marker_trends_index --create-only
```

Verify the generated migration file contains the expected `CREATE INDEX` statements.
