<objective>
Replace the custom `DatabaseRateLimitStore` (which hits Postgres on every request) with `@fastify/rate-limit` v10's built-in Redis store via `ioredis`. This makes rate-limit state persist across deploys and restarts.

Purpose: In-process/database rate limiting loses state on every restart — an attacker can bypass auth rate limits by triggering a deploy. Redis-backed state persists across restarts, satisfying INFRA-03. Using `@fastify/rate-limit`'s built-in Redis integration means atomic sliding-window operations without hand-rolling INCR/EXPIRE logic.

Output: Redis-backed global rate limiter in app.ts, updated rateLimit.ts with ioredis client factory, doppler.yaml updated with new env vars.
</objective>

```typescript
/**
 * Create an ioredis client optimized for @fastify/rate-limit.
 * Returns null if REDIS_HOST is not configured (falls back to in-memory in tests,
 * and DatabaseRateLimitStore in production until Redis is provisioned).
 */
export function createRedisClient(): Redis | null {
  const host = process.env.REDIS_HOST;
  const port = parseInt(process.env.REDIS_PORT || '6379', 10);

  if (!host) return null;

  return new Redis({
    host,
    port,
    connectTimeout: 500,       // @fastify/rate-limit recommendation
    maxRetriesPerRequest: 1,   // @fastify/rate-limit recommendation
    lazyConnect: true,         // Don't connect until first use
  });
}
```

**Step 2: Preserve per-route rate limit configurations**

The per-route limits (auth: 5/15min, phi: 200/min, presign: 50/hr, public: 1000/hr) were previously applied via `createRateLimitMiddleware` hooks in `registerRateLimits`. These now need to move to route-level `config.rateLimit` objects.

However, this requires modifying every route file — which is a large change that could introduce regressions. Instead, use `@fastify/rate-limit`'s `addRateLimitConfig` approach in app.ts to set per-prefix overrides using `onRoute` hook:

```typescript
// Apply per-prefix rate limit overrides after global plugin registration
app.addHook('onRoute', (routeOptions) => {
  const url = routeOptions.url;

  if (url.includes('/auth/login')) {
    // Login rate limiting handled by account lockout middleware (not @fastify/rate-limit)
    // Exclude login from global rate limit to avoid double-counting
    routeOptions.config = {
      ...routeOptions.config,
      rateLimit: { max: 1000, timeWindow: '1 minute' }, // effectively no limit — lockout handles it
    };
  } else if (url.includes('/auth')) {
    routeOptions.config = {
      ...routeOptions.config,
      rateLimit: { max: 5, timeWindow: '15 minutes' },
    };
  } else if (url.includes('/health')) {
    routeOptions.config = {
      ...routeOptions.config,
      rateLimit: { max: 1000, timeWindow: '1 hour' },
    };
  } else if (url.includes('/labs') || url.includes('/photos') || url.includes('/profile')) {
    routeOptions.config = {
      ...routeOptions.config,
      rateLimit: { max: 200, timeWindow: '1 minute' },
    };
  } else if (url.includes('/presign')) {
    routeOptions.config = {
      ...routeOptions.config,
      rateLimit: { max: 50, timeWindow: '1 hour' },
    };
  }
});
```

Place this hook AFTER the `@fastify/rate-limit` plugin registration, BEFORE route registration.
