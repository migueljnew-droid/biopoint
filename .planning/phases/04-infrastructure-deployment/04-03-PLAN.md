<objective>
Harden PHI protection in Datadog logs and Sentry error reports. Extend pino's `redact` paths to cover all healthcare-context PHI fields. Implement `beforeSend` PHI scrubbing in Sentry for breadcrumbs and event context. Reduce `maxBreadcrumbs` from 100 to 20 to limit PHI exposure window. Verify the health check endpoint functions correctly.

Purpose: Logs and error reports are the most common vector for PHI leakage in healthcare apps. The current pino redact list misses key PHI fields used in this codebase (`dateOfBirth`, `value`, `notes`, `markers`). The current Sentry config does header scrubbing but doesn't scrub PHI from request body breadcrumbs or the user.email field. These changes ensure zero PHI reaches Datadog or Sentry even if other scrubbing layers fail.

Output: Updated monitoring.ts with extended redact paths, updated sentry.ts with beforeSend PHI scrubbing.

In `/Users/GRAMMY/biopoint/apps/api/src/config/monitoring.ts`, find the `initializeSentry` function (around line 96) where `Sentry.init({...})` is called. The `beforeSend` hook currently does:

```typescript
beforeSend(event, hint) {
  // Filter out health check errors
  if (event.transaction?.includes('/health')) {
    return null;
  }
  // Add custom context
  event.contexts = { ... };
  return event;
},
```

Replace this `beforeSend` implementation with a PHI-scrubbing version:

```typescript
beforeSend(event, _hint) {
  // Drop health check noise
  if (event.transaction?.includes('/health')) return null;

  // --- PHI SCRUBBING ---
  // 1. Scrub PHI from breadcrumb data (auto-instrumentation captures request/response bodies)
  if (event.breadcrumbs?.values) {
    event.breadcrumbs.values = event.breadcrumbs.values.map(crumb => {
      if (crumb.data) {
        const {
          email, name, value, notes, dateOfBirth, phi, password, token,
          ...safe
        } = crumb.data as Record<string, unknown>;
        crumb.data = safe;
      }
      return crumb;
    });
  }

  // 2. Scrub PHI from extra context (may contain request body snapshots)
  if (event.extra) {
    const {
      email, name, value, notes, dateOfBirth, phi, password, token,
      ...safe
    } = event.extra as Record<string, unknown>;
    event.extra = safe;
  }

  // 3. Strip user.email from Sentry user context — keep only opaque ID
  // User email is a direct identifier and must not be stored in Sentry
  if (event.user?.email) {
    event.user = { id: event.user.id };
  }

  // 4. Add compliance context (non-PHI)
  event.contexts = {
    ...event.contexts,
    app: {
      name: SERVICE_NAME,
      version: SERVICE_VERSION,
      environment: ENV,
    },
    compliance: {
      framework: 'hipaa',
      data_classification: 'phi',
    },
  };

  return event;
},
```
