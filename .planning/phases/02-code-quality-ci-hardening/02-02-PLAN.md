  <name>Task 2: Fix automaticLogoff getStats() bug</name>
  <files>

  <action>
    Fix the bug at line 188 in `getStats()` method.

    **The bug:** `this.isSessionExpired(s.userId)` passes the `userId` to `isSessionExpired()`, but `isSessionExpired()` looks up sessions by session key (the key in `this.sessions` object). Since no session is keyed by userId, it always returns `true` (session not found = expired). This makes `expiredSessions` always equal `totalSessions`.

    **The fix:** Use `Object.entries(this.sessions)` to iterate with both the session key and value:

    ```typescript
    // BEFORE (line 185-188):
    const sessions = Object.values(this.sessions);
    const webSessions = sessions.filter(s => s.deviceType === 'web').length;
    const mobileSessions = sessions.filter(s => s.deviceType === 'mobile').length;
    const expiredSessions = sessions.filter(s => this.isSessionExpired(s.userId)).length;

    // AFTER:
    const sessionEntries = Object.entries(this.sessions);
    const sessions = sessionEntries.map(([, s]) => s);
    const webSessions = sessions.filter(s => s.deviceType === 'web').length;
    const mobileSessions = sessions.filter(s => s.deviceType === 'mobile').length;
    const expiredSessions = sessionEntries.filter(
      ([sessionId]) => this.isSessionExpired(sessionId)
    ).length;
    ```

    This passes the actual session key (the object property key from `this.sessions`) to `isSessionExpired()`, which correctly looks up the session and checks its `lastActivity` timestamp against the timeout.
  </action>

    3. The `webSessions` and `mobileSessions` counts still work correctly (they use the value, not the key).
