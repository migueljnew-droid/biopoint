<objective>
Add Semgrep SAST and npm audit gates to CI so every push is scanned for vulnerabilities.

Purpose: The existing CI has a `security-scan.yml` but it uses the deprecated `returntocorp/semgrep-action@v1` and only runs on pushes to main (not on PRs to develop). The `ci.yml` runs `npm audit --audit-level=moderate` but the requirement specifies `--audit-level=high`. This plan modernizes Semgrep to use the official container approach, ensures both scans run on every push AND PR, and configures them to fail the build on findings.

Output: Every push and PR triggers Semgrep SAST + npm audit. Findings block merges.
</objective>

    The existing `code-security-scan` job (lines 70-119) needs these changes:

    a) Replace the Semgrep step (lines 102-109) with:
    ```yaml
    - name: Run Semgrep SAST scan
      uses: docker://semgrep/semgrep:latest
      with:
        args: >-
          semgrep scan
          --config p/typescript
          --config p/nodejs
          --config p/security-audit
          --config p/secrets
          --error
          --sarif --output semgrep-results.sarif

    ```

    Key changes:
    - Uses `docker://semgrep/semgrep:latest` (official container) instead of deprecated `returntocorp/semgrep-action@v1`
    - `--error` flag makes Semgrep exit with non-zero code on findings (fails CI)
    - `--sarif` outputs SARIF format for GitHub Security tab integration
    - Scans only production directories: `apps/`, `packages/`, `db/`
    - Uses 4 rulesets: `p/typescript`, `p/nodejs`, `p/security-audit`, `p/secrets`
    - Removed `p/react` (API is not React) and `p/javascript` (covered by `p/typescript`)

    b) Add a SARIF upload step right after the Semgrep step:
    ```yaml
    - name: Upload Semgrep SARIF to GitHub Security
      if: always()
      uses: github/codeql-action/upload-sarif@v3
      with:
        sarif_file: semgrep-results.sarif
    ```

    c) Update the artifact upload step (lines 113-119) to include `semgrep-results.sarif`.
