2. Create `apps/api/src/middleware/s3Validation.ts` (NEW FILE):
   - Import `fileTypeFromBuffer` from `file-type`.
   - Import `S3Client`, `GetObjectCommand`, `DeleteObjectCommand` from `@aws-sdk/client-s3`.
   - Define `ALLOWED_MIME_TYPES` as a `Set` containing: `'image/jpeg'`, `'image/png'`, `'image/webp'`, `'application/pdf'`.
   - Export `async function validateUploadedFileType(s3Client: S3Client, bucket: string, key: string)`:
     - Read only the first 4KB from S3 using `Range: 'bytes=0-4095'` header.
     - Pass the buffer to `fileTypeFromBuffer()`.
     - Return `{ valid: boolean; detectedType: string | undefined; declaredType?: string }`.
     - If `!valid`, delete the uploaded object from S3 (cleanup) and log the violation.
   - Export `ALLOWED_MIME_TYPES` for reuse in schemas.

3. Wire validation into the photo upload flow in `apps/api/src/routes/photos.ts`:
   - Find the presigned URL generation endpoint (the one that returns a presigned PUT URL).
   - After the presigned URL is returned to the client AND before the photo record is finalized/confirmed, add a validation step.
   - The typical pattern is: (a) client requests presign URL, (b) client uploads directly to S3, (c) client calls a "confirm upload" or the photo is used later.
   - If there is a "confirm upload" endpoint or a callback, add `validateUploadedFileType()` there.
   - If there is no explicit confirm step, add a note/TODO that validation should occur on the next read of the object.
   - The validation should log a warning if the detected type does not match the declared type.

4. Review `packages/shared/src/schemas/labs.ts` (the `PresignUploadSchema`):
